## Apa yang Dipelajari

- **UI (User Interface):** Tampilan visual yang dilihat dan diklik pengguna.
- **UX (User Experience):** Perasaan dan pengalaman pengguna saat menggunakan produk.
- **Prinsip UI yang baik:** Konsisten, Jelas, Responsif, Estetik.
- **Prinsip UX yang baik:** Mudah dipelajari, Efisien, Tidak membingungkan.
- **Studi kasus:** Membandingkan desain website buruk vs baik.

## Checklist UI yang Baik

```text
✅ Kontras warna cukup (minimal 4.5:1)
✅ Ukuran font minimal 16px untuk body
✅ Button minimal 44x44px (touchable area)
✅ Loading state ditampilkan
✅ Pesan error yang jelas dan membantu
✅ Responsif di semua ukuran layar
```

## Latihan

1. Cari 2 website — satu dengan UI bagus, satu dengan UI buruk.
2. Tulis 5 perbedaan antara keduanya.
3. Identifikasi prinsip UI/UX mana yang dilanggar oleh website buruk.
